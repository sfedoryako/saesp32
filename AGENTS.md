# AGENTS.md

## Описание агентов

Этот файл содержит документацию по агентам, используемым в проекте saesp32.

## Типы агентов

### 1. Системные агенты
- Агенты, отвечающие за базовые системные функции
- Управление ресурсами и конфигурацией

### 2. Пользовательские агенты  
- Агенты для взаимодействия с пользователем
- Обработка пользовательского ввода и команд

### 3. Сетевые агенты
- Агенты для работы с сетевыми протоколами
- Управление сетевыми соединениями

## Конфигурация агентов

Агенты настраиваются через конфигурационные файлы в директории `/config/agents/`.

## Жизненный цикл агентов

1. **Инициализация** - загрузка конфигурации и регистрация агента
2. **Выполнение** - основной цикл работы агента
3. **Завершение** - корректное завершение работы и освобождение ресурсов

## Документация API

Каждый агент должен реализовывать базовый интерфейс:

```cpp
class Agent {
public:
    virtual bool initialize() = 0;
    virtual void run() = 0;
    virtual void shutdown() = 0;
};
```

## Примеры использования

```cpp
// Создание и запуск агента
auto agent = std::make_unique<SystemAgent>();
if (agent->initialize()) {
    agent->run();
    agent->shutdown();
}
```

## Отладка и мониторинг

- Логирование агентов осуществляется через общую систему логов
- Мониторинг состояния агентов доступен через веб-интерфейс
- Метрики производительности сохраняются в базе данных

## Лучшие практики

1. Всегда проверяйте результат инициализации агента
2. Обеспечивайте корректное завершение работы агентов
3. Используйте логирование для отслеживания состояния
4. Документируйте конфигурационные параметры

## Разработка новых агентов

При создании нового агента:
1. Наследуйтесь от базового класса `Agent`
2. Реализуйте все необходимые методы
3. Добавьте конфигурационный файл
4. Обновите документацию
5. Напишите тесты

## Поддержка

По вопросам работы с агентами обращайтесь к команде разработки.
